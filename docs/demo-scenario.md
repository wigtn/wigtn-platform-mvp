# FIELDNOTE 체험용 데모 시나리오 요약

작성 기준: 2026-07-24 20:11 KST  
대상 코드베이스: `/Users/wigtn/wigtn-server/services/fieldnote`

## 1. 데모의 한 줄 정의

FIELDNOTE는 **영업 직군을 위한 회사 리뷰·커뮤니티·AI Q&A·검증/운영 백오피스 입찰 데모**입니다.

평가자는 회원가입 없이 다음 4개 역할을 전환하면서 전체 제품 흐름을 체험합니다.

- 비회원
- 일반 영업인
- 인증 영업인
- 운영 관리자

모든 회사·회원·리뷰 데이터는 합성 데이터이며, 실제 개인정보/실고객 데이터는 사용하지 않습니다.

---

## 2. 역할별 체험 관점

| 역할 | 데모상 의미 | 핵심 체험 기능 |
|---|---|---|
| 비회원 | 가입 전 방문자 | 회사 리뷰 열람, 공개 커뮤니티 열람 |
| 일반 영업인 | 기본 회원 | 프로필 관리, 게시글 작성, 질문 작성, 답변 작성, 배지 신청 |
| 인증 영업인 | 재직/실적 확인된 회원 | 인증 배지가 붙은 리뷰/답변 작성, 신뢰도 높은 활동 |
| 운영 관리자 | 운영팀 | 리뷰 검수, 회원 인증 승인/반려, 콘텐츠 운영, 회사 데이터 관리 |

현재 UI에는 역할 설명이 존재하지만, 권한별 가능/불가능 기능을 한눈에 보여주는 매트릭스는 부족합니다.

---

## 3. 추천 체험 시나리오

### 시나리오 A. 회사 탐색 → 회사 상세 → 익명 리뷰 작성

목적: 회사 리뷰 플랫폼의 기본 가치를 보여줍니다.

흐름:

1. 홈 진입
2. `회사 탐색 시작` 클릭
3. 회사 검색에서 예: `노스스타` 검색
4. `노스스타 클라우드` 상세 페이지 진입
5. `익명 리뷰 작성` 클릭
6. 한 줄 요약, 상세 경험, 6축 점수 입력
7. 리뷰 등록
8. 회사 통계와 평균 점수가 갱신되는지 확인

핵심 메시지:

> 영업인이 목표 현실성, 인센티브 투명성, 리드 품질, 계정 배분, 세일즈 툴, 매니저 코칭 등을 익명으로 평가하고, 회사별 통계로 축적합니다.

---

### 시나리오 B. 영업 Q&A → AI 첫 답변 → 커뮤니티 전환

목적: AI가 영업 상황에 대한 첫 답변 초안을 제공하는 기능을 보여줍니다.

흐름:

1. `/questions/new` 진입
2. 비회원이면 `일반 영업인으로 전환해서 체험하기` 클릭
3. 질문 제목 작성
4. 상황 설명 작성
5. `질문 등록` 클릭
6. `질문 내용 검사` → `AI 초안 작성` → `커뮤니티 답변 대기` 상태 확인
7. AI 답변 카드 확인
8. `커뮤니티에서 보기`로 이동

AI 답변 구조:

- 핵심 판단
- 다음 미팅에서 확인할 질문
- 다음 미팅에서 해볼 일
- 답변을 더 정확하게 만들 정보
- 놓치기 쉬운 점

기술적 의도:

> Supabase 익명 세션 → 방문자별 private queue → 맥미니 worker → AI pipeline guardrail → OpenAI Responses API → polling → 화면 표시

주의:

- Supabase 프론트 설정이 없으면 실제 AI 호출이 아니라 `demo-fallback` 답변으로 동작합니다.
- 현재 UI는 Live/Fallback 여부를 충분히 명확하게 표시하지 않습니다.

---

### 시나리오 C. 커뮤니티 글 작성 → 이미지 첨부 → 상세 확인

목적: 일반 게시판과 파일 첨부 메타데이터 흐름을 보여줍니다.

흐름:

1. `/posts/new` 진입
2. 게시판 선택: 예: `실적`
3. 제목 입력
4. 내용 입력
5. 이미지 첨부
6. 게시글 등록
7. 상세 화면에서 제목과 첨부 이미지 메타데이터 확인

데모 범위:

- 실제 파일 영구 저장보다는 이미지 첨부 메타데이터와 게시글 상태 변화를 보여주는 성격이 강합니다.

---

### 시나리오 D. 커뮤니티 상호작용

목적: FIELDNOTE가 단순 회사 리뷰 저장소가 아니라 영업인 커뮤니티임을 보여줍니다.

체험 기능:

- 글 작성
- 좋아요
- 스크랩
- 댓글
- 답글
- 신고

---

### 시나리오 E. 마이페이지 → 프로필 수정 → 검증 배지 신청

목적: 영업인 신뢰도/인증 체계를 보여줍니다.

흐름:

1. `일반 영업인` 역할로 전환
2. `/account` 진입
3. 표시 이름 수정
4. 경력 한 줄 수정
5. 프로필 저장
6. `샘플 자료로 신청` 클릭
7. 배지 상태가 `검토중`으로 바뀌는지 확인

데모 범위:

- 실제 증빙 업로드/개인정보 수집은 하지 않습니다.
- 합성 증빙 상태만 사용합니다.

---

### 시나리오 F. 인증 영업인 체험

목적: 일반 회원과 인증 회원의 차이를 보여줍니다.

체험 포인트:

- 재직 확인 완료 상태
- 검증 리뷰 작성
- 인증 배지 노출
- 현직자 답변 작성 가능

현재 부족한 점:

- 일반 영업인과 인증 영업인의 차이가 UI에서 충분히 강하게 드러나지 않습니다.

---

### 시나리오 G. 운영 관리자 체험

목적: 서비스 운영팀이 리뷰/회원/콘텐츠/회사 데이터를 관리하는 흐름을 보여줍니다.

흐름:

1. `운영 관리자` 역할 선택
2. `/admin` 진입
3. 리뷰 운영에서 특정 리뷰 블라인드
4. 필요 시 복구
5. 회사 수기 등록
6. XLSX import dry-run
7. 크롤러 후보 dry-run
8. 홈 콘텐츠 배치 관리
9. 회원 인증 승인/반려
10. 콘텐츠 블라인드

관리자 기능 예시:

- 리뷰 블라인드/복구
- 회사 수기 등록
- 샘플 XLSX 파일 검사
- 외부 회사 후보 미리보기
- 홈 추천 회사 게시/게시 취소
- 회원 배지 승인/반려
- 콘텐츠 블라인드

중요한 경계:

- 관리자 기능은 입찰 데모용 harmless command 성격이 강합니다.
- 실제 운영 시스템처럼 보이려면 audit receipt, actor, action id, rollback 가능 여부가 UI에 표시되어야 합니다.

## 4. 크롤링 기능의 현재 의미

크롤링은 최종 사용자 기능이 아니라 **관리자용 회사 데이터 확보/확장 기능 후보**로 언급됩니다.

현재 화면상 위치:

- `/admin/companies`
- 섹션: `외부 회사 후보 미리보기`
- 버튼: `후보 불러오기`
- 결과: `32 신규 후보`, `7 병합 검토`, `0 자동 게시`

의도:

1. 외부에서 회사 후보 수집
2. 출처와 수집 시각 기록
3. 기존 회사와 중복/병합 검토
4. 확인 전에는 공개하지 않음
5. 관리자 검토 후 반입

현재 데모에서는 실제 크롤링을 수행하지 않고 `crawler-dry-run`으로 운영 플로우만 보여줍니다.

## 5. 데모 데이터/저장 방식

### 일반 화면 상태

- 기본은 합성 seed 데이터
- 방문자 액션은 overlay 또는 방문자별 action ledger에 반영
- DB 연결 실패 시에도 시드 데이터 fallback으로 화면이 동작
- 역할 선택 같은 보기 상태는 localStorage에 저장

### AI 질문

- Supabase 설정이 있으면 private queue + worker + OpenAI 경로를 사용
- Supabase 설정이 없으면 `FIELDNOTE_AI_FALLBACK`을 반환
- UI는 이 두 모드의 차이를 명확히 보여주지 못함

---

## 6. 공개 데모에서 의도적으로 하지 않는 것

- 실제 이메일 발송
- 실제 회원가입
- 실제 고객 개인정보 수집
- 실제 업로드 원본 보존
- 실제 크롤링 실행
- 데모 역할 전환을 production 인증으로 재사용

이 항목들은 빠진 기능이라기보다, 개인정보/외부 부작용을 피하기 위해 합성 adapter로 대체한 경계입니다.

---

## 7. 발표용 한 줄 요약

FIELDNOTE 데모는 평가자가 가입 없이 비회원, 일반 영업인, 인증 영업인, 운영 관리자 역할을 오가며 회사 리뷰 탐색, 익명 리뷰 작성, 영업 Q&A와 AI 첫 답변, 커뮤니티 활동, 프로필/검증 배지, 관리자 검수까지 한 번에 체험하는 영업 커리어 인텔리전스 MVP입니다. 실제 개인정보와 고객 데이터는 쓰지 않고, 합성 데이터와 방문자별 격리된 데모 상태로 전체 제품 흐름을 보여줍니다.
